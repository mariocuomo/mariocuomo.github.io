Stale Account Auditor

Finds accounts that haven't signed in for 90+ days, privileged accounts with no recent activity, and unused applications. Recommends disabling or reviewing stale identities.

Identity Security Entra ID KQL Defender for Identity

Skills Used

GetEntraData GetEntraUserDetailsV1 GetEntraUnusedApplication GetDefenderIdentitySummary GetStalePrivilegedLogonActivity

Typical Workflow

  1. Stale User Accounts — Uses Entra ID to list all users with no sign-ins in the last 90 days, including UPN, display name, and account status.
  2. Privileged Account Review — Queries Entra ID for all users with high-privilege directory roles (Global Admin, Security Admin, etc.) and identifies those inactive for 30+ days.
  3. Defender Identity Cross-Check — Runs a KQL query against IdentityLogonEvents to verify that flagged privileged accounts have no on-premises logon activity either.
  4. Unused Applications — Uses the built-in Entra skill to find applications that haven't issued tokens in 90+ days.
  5. Stale Guest Accounts — Queries Entra ID for guest users with no sign-in activity in 90+ days.
  6. Report Generation — Consolidates all findings into a structured audit report with summary, tables per category, and prioritized remediation recommendations.
