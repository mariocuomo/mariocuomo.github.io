Endpoint Health Monitor

Reports on unhealthy, offline, or misconfigured Defender for Endpoint agents, onboarding gaps, and sensor health issues across the fleet.

Endpoint Security KQL

Skills Used

GetSensorHealthOverview GetUnhealthyDevices GetOnboardingGaps GetSecurityMisconfigurations

Typical Workflow

  1. Sensor Health Overview — Queries DeviceInfo for a breakdown of all devices by SensorHealthState (Active, Inactive, ImpairedCommunication, NoSensorData).
  2. Unhealthy & Offline Devices — Retrieves the list of devices whose sensor is not Active, with device name, OS, last seen timestamp, and exposure level.
  3. Onboarding Gaps — Identifies devices discovered in the network but not yet onboarded to Defender for Endpoint.
  4. Security Misconfigurations — Finds devices with non-compliant security configurations that have a high impact score (≥ 7).
  5. Report Generation — Consolidates all findings into a structured Endpoint Health Report with fleet summary, tables, and prioritized recommendations.
